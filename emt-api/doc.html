Introduction EMT H2H API

Overview of Integration API
Base URL
  • https://api.ptx.xyz (Production Server Live Data)
  • https://api-sbx.ptx.xyz (Sandbox Server Test Data)
Request
Method Endpoint Usage
POST /auth/token Get Access Token (auth_token) for Oauth2
POST /auth/refresh Get a new auth_token
POST /topup Send topup request
GET /topup/{slug} Get Topup Detail By Slug
GET /topup/?ref_code={ref_code} Get Detail of Specific Topup by using its reference Code when using ref_code if more than one Record is found only the latest one will be returned
GET /balance Get wallet data
GET /history/topup Topup request history data
GET /history/wallet Successful transaction data history/entered balance history

Authentication Oauth2 (Access Token)

Access Token is used to identify a user or device.
Request
Setting Value
HTTP Method POST
Path /auth/token
Host https://api-sbx.ptx.xyz
Request Headers
Name Format Mandatory Description
Authorization Basic base64(client_id:client_secret) Yes
Content-Type application/json Yes
Request
Name Format Mandatory Description
grant_type String Yes Value = client_credentials
Result of the request will contain the following information :
Response
Field DataType Description
access_token String Your access_token
token_type String Default is Bearer
expires_in String access_token validity, in x-seconds
refresh_token String Used instead of client_id:client_secret but is one timeuse to get new auth_token
scope String Application scope / permission granted to application
